## 求关键路径、最早开始时间、最晚开始时间

![图的应用_2025-12-10_16-41-48.png|400](/img/user/attachments/%E5%9B%BE%E7%9A%84%E5%BA%94%E7%94%A8_2025-12-10_16-41-48.png)
1. 先找最长的路径 `1->3->4->6`，对应的边是 `b->e->i` 总长度为 12，最长的路径就是关键路径.
2. 求每个节点的最早开始时间，口诀为：某个边的源点之前的能完成的最早，比如 3 这个节点对应着 e、g、f 三个边，那么 e、g、f 的最早开始=到达状态 3 最晚时间. 因此看 e、g、f 的最早开始只需要看指向 3 的 b 和 c 的完成时间.
3. 求每个节点的最晚开始时间，口诀为：某个边的最晚开始=之前的最晚开始-本边用时，例如看 d 和 e 都是 4 号节点，他们的最晚开始等于 4 号节点的最晚开始，4 号节点最晚开始等于 5、6 的最晚开始分别减去 h 和 i 的时间。

| 序号  | 最早开始时间 (按照依赖顺序，从左往右找） | 最晚开始时间（从右往左回推） | 活动余量 | 是否为关键路径 |
| --- | --------------------- | -------------- | ---- | ------- |
| a   | 0                     | 2              | 2    |         |
| b   | 0                     | 0              | 0    | 是       |
| c   | 2                     | 4              | 2    |         |
| d   | 2                     | 5              | 3    |         |
| e   | 5                     | 5              | 0    | 是       |
| f   | 5                     | 10             | 5    |         |
| g   | 5                     | 11             | 6    |         |
| h   | 8                     | 10             | 2    |         |
| i   | 8                     | 8              | 0    | 是       |
| j   | 9                     | 11             | 2    |         |

其他知识点：
1. 关键路径是从源点到汇点路径路径长度最长的路径
2. 增加关键路径上的任意一个活动一定会延长工期
3. 缩短关键路径上的一个活动不一定会减少工程工期，只有全部关键路径都减少时候，才会减少.
## 迪杰斯特拉求解过程
算法思想：贪心算法，每次都维护到其他的最短距离.

用途：求单源最短路径
![图的应用_2025-12-10_18-31-54.png|200](/img/user/attachments/%E5%9B%BE%E7%9A%84%E5%BA%94%E7%94%A8_2025-12-10_18-31-54.png)

求 1 到其余各顶点的求解过程：

| 节点                  | 初始           | 第一轮            | 第二轮         | 第三轮         |
| ------------------- | ------------ | -------------- | ----------- | ----------- |
| 2                   | 26           | 25             | 21          | 15          |
| 3                   | 3            | -（上一轮已加入，无需更新） | -           | -           |
| 4                   | ♾️           | ♾️             | 14          | -           |
| 5                   | 6            | 6              | -           | -           |
| dist{}，保存到其他节点的最短距离 | {26,3, 无穷，6} | {25,3, 无穷, 6}  | {21,3,14,6} | {15,3,14,6} |
| 本轮选择                | 3 号节点        | 5 号节点          | 4 号节点       | 2 号节点       |
| 已求解节点               | 1、3          | 1、3、5          | 1、3、5、4     | 1、3、4、5、2   |

其他知识点
* 适用于非负权重的有向图或者无向图。
* 与 Prim 很像，Prim 是加点，每次比较是新加入的点跟随的边的权重，迪杰斯特拉则是比较新加入的点到源点的总路径长度.
