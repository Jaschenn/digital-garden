## 基于BST的查找
（Binary Search Tree）也叫二叉查找树，优化之后的就是平衡二叉树
* 前提条件：数据已经使用 BST/AVL/红黑树存储了。
#### 查找过程
1. 从根节点开始查找
2. 如果当前节点等于目标值，返回当前节点
3. 如果当前节点的值小于目标节点，在右子树进行查找
4. 如果当前节点的值大于目标节点，在当前节点的左子树进行查找。
5. 如果到达空节点，则查找失败，返回 NULL。

| 树类型 | 平均时间复杂度 | 最坏时间复杂度                              |
| --- | ------- | ------------------------------------ |
| BST | log n   | n(BST 没有对树的形状做要求，因此最坏情况下，树可能退化成一个链表) |
| AVL | log n   | log n                                |
| 红黑树 | log n   | log n                                |
## 平衡二叉树
#### 平衡二叉树
* 任意节点的左右子树高度差不超过 1 
* 平衡二叉树构造过程: 关键
	* 找到三个要调整的节点
		* 从最深的叶子结点向上找第一个失衡的点，标号为 1
		* 再选 1 下面的点为 2，再选 2 下面的点为 3
	* 进行调整
		* 按照节点值大小关系（不是 123 的编号），构成新的稳定三角结构
		* 剩下的左边在左边右边在右边

## B-树（也叫 B 树）
（多路平衡查找树，用于数据库和文件系统，存储、检索有序数据）
#### 特点
* 多路：对比 BST只是二叉树，每个节点最多只有两个孩子，B-树每个节点可以有多个孩子，最多的孩子数量就是 B-树的阶。孩子的数量等于关键字的数量+1.
* 平衡：B-树所有的叶子节点在同一层，防止树像BST 一样退化成链表，确保树的平衡，保证查找效率。==注意并不是所有的关键字全部都存储在叶子节点上，根节点、以及中间的结点上存有数据，这是和 B+树不一致的地方. ==
---
* 对比 AVL，m 阶B-树通过限制子树的关系来尽可能的降低层的高度，以提高查找效率。
	* 【通用限制】每个结点最多有 m 棵子树，最多有 m-1 个关键字。
	* 对根节点的限制
		* 若根结点不是叶子结点，则至少有两个子树（即根节点至少有一个关键字）
	* 对其他非叶节点
		* 除了根节点之外，所有的非叶结点最少有 $\lceil {m/2} \rceil$ 棵子树。例如 3 阶 B 树，至少有 2 棵子树，即至少有一个关键字；5 阶 B 树，至少有 3 个子树，即两个关键字.
