# CS 和 OS 思维导图

## 2016
- 三对角阵的压缩存储
- B+树和 B 树的不同点：B+树叶子节点之间按照从小到达的顺序链接在一起，可以支持顺序查找
- 高级语言翻译成目标代码的程序是：编译程序。（现代编译程序有两种，一种是先翻译成汇编，再翻译成机器目标代码，还有一种是直接翻译成目标代码）
- -32767 的补码机器数为 1000 0000 0000 0001
- a[k] = a[k] + 32,计算 Cache 缺失率，太他妈坑了
- 单周期处理器不能用单总线，因此一次要处理多个操作数。
- 访存时缺页，属于异常！ 虽然叫缺页中断，但是是当前程序执行中发生的，与当前指令相关，估是 page default 异常。
- 批处理系统是用户的作业排队，由作业调度逐一调入内存，用不不能够直接与计算机进行交互。
- 改进型 clock 算法的淘汰次序为：先按访问位，再按修改位。
- 工作集的概念，最近n 个去重 vs 驻留集
- Spooling 技术：将信息放在磁盘上，然后排队处理，可以将设备改造成过个作业共享的。例如打印机。

## 2022
- 补码表示的数据范围
- float 真值和机器数表示
- DRAM 行列地址线复用
- 指令系统条数 设计 #2022
- 高级语言-> 可执行文件的过程：预处理->编译->汇编->链接
- 系统调用过程，os 完成的是 保存通用寄存器的内容，执行系统调用服务例程，没有保存断点和程序状态字（这些是硬件完成，也没有更改 CPU 模式为内核态，这是由硬件逻辑控制的）

## 2023
- CPI 计算
- Short 变量 -8190  真值转机器数
- Float 的机器数转真值
- CF/OF 标记判断
- 中断相关：外部设备通过中断控制器向 CPU 发中断结束信号，正确的应该是 CPU 给中断控制器发，设备根本不知道中断有没有处理完，只有 CPU 才知道，执行中断响应程序之后就会自动退出中断，不需要设备请求退出。

## 2024
- 有符号数： int 转 short，直接截断，short 转int，符号拓展
- 主存和外存之间一般采用命中率较高的全相联映射，主存到 Cache 之间一般用直接映射。 远远分别是不命中代码高，以及为了更快。
- MMU 地主转换过程中不检测：Cache 缺失，地址翻译的时候都还没有 Cache 的事情呢
- 中断IO 控制方式，错误叙述：中断屏蔽字用于确定中断影响的优先级【错误，只能确定中断的开关，无法确定优先级】
	- 处理流程：保存断点和程序状态字在中断响应阶段由硬件完成，保存通用寄存器和设置新的中断屏蔽字由 OS 软件完成
- 流水线的冒险相关
	- load use 型的数据冒险通过旁路（转发电路）无法解决
- 总线传输速率，计算
- 关于中断的叙述
	- 中断或异常发生的时候，CPU 处于内核态【错误】
	- 每个系统调用都有对应的内核服务例程
	- 中断处理程序开始执行的时，CPU 处于内核态
	- 系统添加新类型的设备时，需要注册对应的中断服务例程
- OS 在终止进程的时候
	- 回收进程占用的设备
	- 释放进程控制块 pcb
	- 回收为进程分配的内存
	- 终止子进程【错误】Linux 中有专门的系统级进程负责管理这种子进程

## 2025
